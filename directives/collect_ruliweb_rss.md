# 루리웹 유머게시판 RSS 키워드 본문, 이미지 및 댓글 수집 지침 (SOP)

이 지침서는 루리웹 유머게시판 RSS 피드를 모니터링하여 특정 키워드가 포함된 게시글의 상세 본문, 이미지 파일, 그리고 추천수 5개 이상인 댓글을 함께 수집하는 자동화 시스템의 운영 방법을 정의합니다.

## 1. 개요 및 목적
1. 루리웹 유머게시판(`https://bbs.ruliweb.com/community/board/300143/rss`)에서 사용자가 지정한 관심 키워드(예: 버튜버, 붉사, 붉은사막 등)가 제목에 포함된 글을 감지합니다.
2. 매칭되는 글의 상세 링크로 접속하여 웹 크롤링을 통해 게시글의 전체 본문 텍스트를 추출합니다.
3. 본문에 포함된 **이미지 파일**들을 자동으로 감지하여 다운로드하고, 보고서 파일명과 동일한 접두사를 가진 `.jpg` 파일로 저장합니다.
4. 해당 글에 등록된 댓글 중 **추천수가 5개 이상**인 베스트 댓글들을 자동으로 필터링하여 수집합니다.
5. 수집된 본문 내용, 다운로드된 이미지 링크(마크다운 내 인라인 렌더링), 베스트 댓글 목록을 마크다운(`.md`) 파일로 작성해 저장합니다.

## 2. 사전 준비 및 설정
수집 동작은 `.env` 파일을 통해 구성할 수 있습니다.

### 환경 변수 설정 (`.env`)
`.env` 파일에 아래 설정을 입력하여 기본 동작 방식을 변경할 수 있습니다:
```env
# 수집할 키워드 목록 (쉼표로 구분)
RSS_KEYWORDS=버튜버,붉사,붉은사막

# RSS 폴링 주기 (초 단위, 기본값: 300초 = 5분)
RSS_INTERVAL=300

# 수집 실행 시간 (시간 단위, 기본값: 24시간)
RSS_DURATION=24

# 결과 저장 파일 경로
RSS_OUTPUT_PATH=collected_posts.csv

# 결과 리포트 및 이미지를 저장할 디렉터리 경로
RSS_ANALYSIS_DIR=ruli-analysis

# 베스트 댓글 선정 기준 추천수 (기본값: 5)
RSS_LIKE_THRESHOLD=5
```

## 3. 실행 방법

### 일반 실행 (포그라운드)
콘솔 창에서 아래 명령어를 직접 실행합니다.
```bash
python execution/collect_ruliweb_rss.py
```

명령줄 파라미터를 사용해 동적으로 키워드와 설정을 입력하여 실행할 수도 있습니다:
```bash
python execution/collect_ruliweb_rss.py --keywords "붉은사막" --likes 3 --duration 12 --interval 120
```

### 백그라운드 실행 (Windows PowerShell)
24시간 동안 중단 없이 백그라운드에서 실행하려면 아래 명령어를 사용합니다.
```powershell
Start-Process python -ArgumentList "execution/collect_ruliweb_rss.py" -WindowStyle Hidden
```

## 4. 출력 및 상태 관리
- **전체 수집 메타데이터**: `collected_posts.csv`에 한글이 깨지지 않도록 `UTF-8 with BOM` 형식으로 누적 기록됩니다.
- **수집 리포트 및 이미지**: `ruli-analysis/` 폴더 하위에 마크다운 및 이미지 파일로 저장됩니다.
  - 리포트 파일명 형식: `ruli-analysis/report_YYYYMMDD_HHMMSS_{post_id}.md`
  - 이미지 파일명 형식: 1개인 경우 `report_..._{post_id}.jpg`, 여러 개인 경우 `report_..._{post_id}_1.jpg`, `report_..._{post_id}_2.jpg` 등
  - 마크다운 내용 내부에 다운로드한 로컬 이미지가 **인라인 이미지 태그**(`![이미지](...)`)로 포함되어 있어, 마크다운 뷰어로 열었을 때 이미지가 자동으로 출력됩니다.
- **중복 방지 상태**: `.tmp/rss_state.json`에 이미 수집 및 필터링 완료된 게시물의 링크가 영속적으로 저장되어 중복 처리를 방지합니다.

## 5. 문제 해결 (Self-annealing & Edge Cases)
- **네트워크 연결 오류**: RSS 피드 수신 실패나 상세 본문 HTML 호출에 실패하더라도 수집기는 멈추지 않고 작동을 유지합니다.
- **이미지 다운로드 실패**: 특정 이미지 서버 오류나 트래픽 제한으로 다운로드가 실패하는 경우, 로그에 경고를 남기고 다음 이미지 수집을 계속 진행하며, 마크다운 리포트는 문제없이 생성됩니다.
